Top priorities

Ordered by impact × effort.

  1. Fix the limited-connection crash (#1). Guard updateElementOptions on result.isRegistered at all five call sites. Highest impact — a hard crash for every data-saver / slow-connection user — and invisible to the current test suite.

  4. Fix the meta key and deep-watch gaps (#3, #5). Small, contained correctness fixes.

Method & trust notes

How these findings were produced and validated.

Multi-agent verification. 9 finder agents swept distinct dimensions (each React/Vue hook, the directive, types/DX, parity, leaks/async). Every candidate finding was handed to an adversarial verifier instructed to refute it against source; a completeness critic then swept for misses. 54 raw → 51 survived verification → +4 from the critic.
Test-proven bugs. Two bugs were reproduced with executable tests against the real hook code: bug #1 (React useForesight crashes on mount on a limited connection; useForesights crashes on the first option change, not mount), and bug #3 (meta-only change → 0 patch calls vs 1 for a name change).
Correction applied. A verifier over-rated the React useForesights limited-connection bug as a mount crash. My own test (MULTI_THREW=no) proves the multi hook does not crash on mount — its patch effect deps are [length, patchKey], which don’t change when refs merely attach. The crash is deferred to the first option change. Severity adjusted accordingly.

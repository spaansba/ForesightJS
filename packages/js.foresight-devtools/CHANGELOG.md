# js.foresight-devtools

## 2.2.1

### Patch Changes

- [#167](https://github.com/spaansba/ForesightJS/pull/167) [`21f066a`](https://github.com/spaansba/ForesightJS/commit/21f066a7be24bb111dd8e2d87208598c6e358edd) Thanks [@spaansba](https://github.com/spaansba)! - Store event listeners in a `Set` instead of an array. Iterating a `Set` during `emit` means a listener that unsubscribes another mid-dispatch no longer causes the next listener to be skipped, and registration now dedupes like the DOM `addEventListener`.

- [#172](https://github.com/spaansba/ForesightJS/pull/172) [`17469e6`](https://github.com/spaansba/ForesightJS/commit/17469e6687dc37a7cfd9a0139b4b6e1cf646d199) Thanks [@spaansba](https://github.com/spaansba)! - Element overlays no longer force a compositor layer per overlay (`will-change: transform`) and name labels no longer use `backdrop-filter: blur`. On pages with many visible elements this kept hundreds of GPU layers alive and re-sampled a blur every frame, saturating the compositor and making the mouse trajectory line visibly lag behind the cursor. Labels now use a more opaque background so they stay readable without the blur.
